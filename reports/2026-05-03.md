2. **研究 SGLang 中 UnifiedRadixTree 的 HiCache 框架支持，分析其对设备/主机缓存生命周期的统一管理**：今日 [sgl-project/sglang [PR #23316](https://github.com/sgl-project/sglang/pull/23316)] 为 UnifiedRadixTree 添加了 HiCache 框架支持，统一了设备/主机缓存生命周期。这与你关注的 "prefix-caching" 和 "KV Cache 管理" 高度相关。建议你阅读该 PR 的代码变更，重点关注 HiCache 如何管理缓存的生命周期（如缓存分配、释放、迁移策略）。然后，尝试在 SGLang 中，使用一个长序列推理场景（如 32K tokens），对比启用/禁用 HiCache 时的缓存命中率和端到端延迟，并分析其对 DeepSeek 等 DSA 模型的适用性。

3. **评估 vLLM 中 FP8 Bias 加载修复对量化模型推理正确性的影响**：今日 [vllm-project/vllm [PR #41424](https://github.com/vllm-project/vllm/pull/41424)] 修复了 FP8 量化模型中 bias 加载时因元张量物化导致 bias 值被覆盖为随机值的问题。这与你关注的 "量化（FP8）" 方向直接相关。建议你阅读该 PR 的代码变更，重点关注 `load_bias` 函数中如何避免元张量物化。然后，在本地 vLLM 环境中，使用一个 FP8 量化模型（如 Qwen3-30B-A3B-Thinking-2507-FP8），对比修复前后在相同输入下的输出 logits 一致性，验证该修复对推理正确性的影响。

4. **分析 vLLM 中 flashinfer 自动调优被禁用的根因，评估其对推理性能的潜在影响**：今日 [vllm-project/vllm [PR #41524](https://github.com/vllm-project/vllm/pull/41524)] 临时禁用了 flashinfer 的自动调优功能，因为发现了正确性问题。这与你关注的 "性能优化" 和 "推理系统" 方向相关。建议你阅读该 PR 的讨论和代码变更，重点关注自动调优在哪些场景下触发了正确性 bug（如 O1/O2 级别）。然后，在本地 vLLM 环境中，使用一个支持 flashinfer 的模型（如 LLaMA-3-8B），对比启用/禁用自动调优时的解码吞吐和输出一致性，评估该修复对生产环境部署的潜在影响。
