4. **分析 FlashInfer 中 GDN 注意力 scale=0.0 的数值一致性问题**：今日 [commit](https://github.com/flashinfer-ai/flashinfer/commit/...) 修复了 GDN 注意力中 scale=0.0 在不同 GPU 架构上的数值不一致问题，并优化了 SM100 路径的内存分配。这与你“性能优化”和“CUDA kernel 开发”方向相关。建议你阅读该 commit 的代码变更，理解其如何通过调整计算顺序或使用特定指令来保证跨架构的数值一致性。你可以将此修复作为案例，分析在你的 CUDA kernel 开发中，当涉及跨代 GPU（如 Hopper 与 Blackwell）时，可能遇到的类似数值精度问题，并提前设计测试用例。

5. **跟进 SGLang HiCache 在上下文并行场景下的同步修复**：今日 [[PR #20460](https://github.com/sgl-project/sglang/pull/20460)](https://github.com/sgl-project/sglang/pull/20460) 为 HiCache 添加了上下文并行（CP）下的同步支持，确保不同 CP 等级在预取完成/撤销、写回确认和主机缓存更新上保持一致。这与你“分布式推理”和“KV Cache 管理”方向高度相关。建议你阅读该 PR 的代码变更，重点关注其如何扩展同步范围从 `tp_group` 到 `attn_cp` 和 `attn_tp` 组。然后，尝试在本地 SGLang 环境中，使用一个长上下文模型（如 LLaMA-3-8B-128K），对比启用 HiCache 前后在 CP 场景下的缓存命中率与端到端延迟。
