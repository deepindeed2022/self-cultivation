2.  **分析 transformers 中 rotary_emb 重复计算的性能优化**：今日的 PR #45555 “perf: avoid recomputing rotary_emb for each layer in some Google and ModernBERT models” 针对 BERT 类模型优化了位置编码计算，这与你“CV模型/BERT模型等在新硬件上的优化方案”高度相关。建议你阅读该 PR 的代码变更，理解其如何通过缓存或共享 `rotary_emb` 来避免逐层重复计算。你可以尝试将此优化思路移植或对比到你熟悉的推理框架（如 TensorRT-LLM 或 vLLM）中处理类似模型（如 ModernBERT）的场景，评估其在 Blackwell 等新硬件上可能带来的性能收益。

3.  **研究 NVIDIA Model-Optimizer 中小维度 Matmul 节点的 INT8 量化排除策略**：今日的 PR #1256 “Exclude small-k and small-n Matmul nodes from Int8 quantization” 直接关联你的“量化（W4A16/FP8）”研究方向。建议你分析此 PR 的改动，理解其判定“小维度”的具体阈值和避免性能回归的逻辑。你可以将此策略与你关注的 FP8/FP4 量化方案进行对比，思考在 Blackwell 等支持新数据类型的硬件上，是否需要对不同精度（INT8 vs. FP8）和不同算子形状采取差异化的量化策略，并设计实验进行验证。

4.  **关注 vLLM 中 TritonExperts 对 FP8 支持的通用化后端适配**：今日的 PR #40132 “[xpu][rocm] Update `current_platform.supports_fp8()` for TritonExperts” 虽然主要针对非 CUDA 后端，但其“使逻辑更通用”的改动体现了推理框架对多硬件平台 FP8 支持的统一抽象。建议你查看此函数的具体实现，理解其如何检测硬件对 FP8 的支持。这有助于你评估在 Blackwell 等新硬件上部署时，vLLM 等框架的 FP8 推理路径的成熟度和可能需要适配的环节。
