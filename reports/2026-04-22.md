1.  **跟进 SGLang 对 LFM2 MoE 模型的调优支持**：今日的 PR #22791 “[MoE] Add LFM2 MoE tuning support + tuned configs for H100/B200/MI325X” 直接关联你的“MoE dispatch”和“性能优化”方向，并涉及 B200 等新硬件。建议你仔细阅读该 PR 中发布的针对不同 GPU 和 TP 设置的优化内核配置，理解其调优方法。你可以尝试在本地 SGLang 环境中，使用提供的配置对 LFM2 MoE 模型进行基准测试，并与默认配置对比，验证其高达 47% 的吞吐量提升，并思考其调优策略是否可迁移至你关注的其他推理引擎（如 vLLM 或 TensorRT-LLM）的 MoE 支持中。

2.  **研究 SGLang 中 MambaPool 的 KV Cache 卸载机制**：今日的 PR #22493 “Add MambaPool kvcache offloading during retraction” 为 Mamba 模型的 SSM 状态实现了 CPU 卸载功能，这与你关注的“KV Cache 管理”和“内存优化”高度相关。建议你阅读此 PR 的代码，重点关注其如何在请求回退（retraction）时保存和恢复状态。你可以分析其设计，并与传统 Transformer 的 PagedAttention KV 卸载方案进行对比，评估这种针对状态空间模型（SSM）的专用卸载策略的优缺点及其对系统设计的影响。

3.  **分析 vLLM 中多模态推理的预热优化**：今日的 PR #40409 “[Bugfix] avoid warmup if text only expectation in multi_modal run” 修复了在多模态运行时，当仅期望文本输入时仍进行不必要预热的问题。这关联你的“性能优化”和“工业界先进的推理方案”方向。建议你阅读该修复的代码，理解其判断逻辑。你可以基于此，思考在更通用的多模态推理服务中，如何根据输入模态动态调整预热和资源分配策略，以进一步优化端到端延迟和资源利用率。

4.  **验证 vLLM 中 fused_topk_bias 对 NaN/Inf 的处理**：今日的 PR #40553 “test: add nan/inf clamp regression test for fused_topk_bias” 为确保 MoE 模型中专家选择的正确性添加了回归测试。这与你关注的“MoE”和“推理系统稳定性”相关。建议你查看新增的测试用例，理解在何种场景下（如 DeepSeek 风格模型）fused_topk_bias 可能产生 NaN/Inf 值以及 clamp 操作的必要性。你可以在本地 vLLM 环境中运行此测试，并思考类似的数值稳定性问题是否可能出现在你正在调研的其他低精度（如 FP4）推理方案中。
